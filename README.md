# 📁 Формат входных данных

Программа работает с JSON-файлами следующей структуры:

```json
{
  "objects": ["кошка", "собака", "рыба"],
  "attributes": ["млекопитающее", "хищник", "плавает", "домашнее"],
  "context": [
    [1, 1, 0, 1],
    [1, 1, 0, 0],
    [0, 1, 1, 0]
  ]
}
```

### Где:

* `objects` — список объектов
* `attributes` — список признаков
* `context` — матрица инцидентности

  * `1` — объект обладает признаком
  * `0` — не обладает

# 📋 Команды

| Команда               | Описание                              | Пример                           |
| --------------------- | ------------------------------------- | -------------------------------- |
| `help`                | Показать справку                      | `help`                           |
| `info`                | Информация о загруженном контексте    | `info`                           |
| `load_context <путь>` | Загрузить контекст                    | `load_context data/example.json` |
| `build_set`           | Построить понятия (set-реализация)    | `build_set`                      |
| `build_bitset`        | Построить понятия (bitset-реализация) | `build_bitset`                   |
| `compare`             | Сравнить производительность           | `compare`                        |
| `list [страница]`     | Список понятий (по 10 на странице)    | `list` или `list 2`              |
| `show <номер>`        | Детали понятия                        | `show 1`                         |
| `export [путь]`       | Экспорт результатов                   | `export results.json`            |
| `exit / quit`         | Выход                                 | `exit`                           |

# ⚡ Сравнение производительности

| Реализация     | Структура               | Скорость          | Память | Когда использовать               |
| -------------- | ----------------------- | ----------------- | ------ | -------------------------------- |
| `build_set`    | `set`                   | Медленнее         | Больше | Малые контексты, отладка         |
| `build_bitset` | `int` (битовые вектора) | Быстрее в 2–5 раз | Меньше | Большие контексты (до 1000×1000) |

**Примечание:**
Bitset-реализация использует побитовые операции, что дает существенный прирост производительности.

# 📚 Теоретическая справка

## Формальный контекст

Тройка **(G, M, I)**, где:

* **G** — множество объектов
* **M** — множество признаков
* **I ⊆ G × M** — отношение инцидентности

---

## Формальное понятие

Пара **(A, B)**, где:

* **A ⊆ G** — объем (extent)
* **B ⊆ M** — содержание (intent)

Условия:

```
A' = B
B' = A
```

---

## Алгоритм Van Der Merwe, Kourie (2002)

Рекурсивный алгоритм построения всех формальных понятий:

1. Начинаем с нижнего понятия
2. Генерируем новые, добавляя признаки
3. Проверяем уникальность
4. Продолжаем рекурсивно

# 🐛 Возможные ошибки и решения

| Ошибка                         | Причина                 | Решение                  |
| ------------------------------ | ----------------------- | ------------------------ |
| Файл не найден                 | Неверный путь           | Проверьте путь           |
| Отсутствуют обязательные ключи | Неверный JSON           | Проверьте структуру      |
| Сначала загрузите контекст     | Контекст не загружен    | Выполните `load_context` |
| Нет результатов для экспорта   | Не выполнено построение | Выполните `build_set`    |
| Значение 2 в матрице           | Некорректные данные     | Используйте 0 и 1        |
| Список объектов пуст           | Пустой JSON             | Добавьте объекты         |
| Объект 'X' не найден           | Несуществующий объект   | Проверьте список         |
